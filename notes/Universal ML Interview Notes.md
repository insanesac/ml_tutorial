# Universal ML Interview Notes

## Shape Rules

Always ask:
- N = samples
- D = features
- C = classes

## Complexity Rules

DSA: O(N)

ML: Often O(ND) because each sample has D features.

## Bias vs Variance

### High Bias
- Too simple.
- Misses patterns.
- Underfitting.

### High Variance
- Too flexible.
- Learns noise.
- Overfitting.

## Daily Oral Drill

For every algorithm answer:
1. What problem does it solve?
2. Supervised or unsupervised?
3. Loss function?
4. Complexity?
5. Biggest weakness?
6. One improvement?

If you can answer those six questions quickly, you're usually interview-ready on that topic.

These are the notes I'd personally want in front of me 30 minutes before a Google ML coding interview.
